# Веб-сервис инференса моделей

## 1. Подготовка окружения и установка зависимостей

Для обеспечения корректной работы и изоляции пакетов рекомендуется использовать виртуальное окружение Python.

Перед началом работы необходимо выполнить следующие действия:

1.  Создать виртуальное окружение:

        python -m venv venv


2.  Установить зависимости, указанные в файле `requirements.txt`:

        pip install -r requirements.txt

## 2. Архитектура и принцип работы приложения

Приложение разделено на backend и frontend составляющие.

**backend** реализован с использованием паттерна управления состояниями и функционирует по циклу `load-use-unload` с привлечением специализированных вычислительных модулей (engines). 

Важной особенностью реализации является следующий механизм инициализации ресурсов:
-   При отсутствии необходимых файлов модели на локальном диске запускается процедура обращения к удаленному репозиторию Hugging Face.
-   Производится автоматическая загрузка недостающих компонентов, необходимых для корректного выполнения инференса.

**frontend** предоставляет веб-интерфейс для взаимодействия с системой.

## 3. Запуск веб-сервиса

В качестве веб-сервера для развертывания сервиса используется фреймворк Flask.

Для запуска приложения необходимо выполнить скрипт `main.py`:

        python main.py
        по пути - 
        /ml-lab/InferenceModelApp/main.py

так же существует возможность запуска решения в контейнере:

        sudo docker build -t my-app .
        по пути - 
        /ml-lab/InferenceModelApp/Dockerfile

        рекомендуется игнорировать кеш:
        sudo docker build --no-cache -t my-app .


После успешного запуска в консоли будет отображен сетевой адрес, предоставляемый сервером Flask. Перейдя по данному адресу в браузере, пользователь получает доступ к веб-интерфейсу для выполнения операций инференса.

## 4. Примечание о режиме вычислений

С целью обеспечения максимальной воспроизводимости результатов и устойчивости работы на различных аппаратных конфигурациях, выполнение вычислений и загрузка моделей принудительно ограничены центральным процессором. Данное ограничение введено осознанно в целях повышения надежности, пусть и в ущерб скорости обработки данных. Такой подход гарантирует стабильность окружения вне зависимости от наличия или версии дискретного графического адаптера.
